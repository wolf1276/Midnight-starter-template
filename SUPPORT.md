# Support

## Getting help with this starter template

- **Environment/setup issues:** run `npm run doctor` first — it diagnoses the most common
  problems (missing prerequisites, stopped Docker services, unbuilt artifacts) and tells you
  the exact command to fix each one.
- **"Why does X work this way?":** see [docs/WHAT_IS_MIDNIGHT.md](docs/WHAT_IS_MIDNIGHT.md) for
  the conceptual background, or [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) for how this repo
  is put together.
- **Something's broken:** check [docs/TROUBLESHOOTING.md](docs/TROUBLESHOOTING.md) for known
  issues and fixes.
- **Bug reports or feature requests:** open an issue in this repository, or check
  [CONTRIBUTING.md](CONTRIBUTING.md) for how to submit one.

## Getting help with Midnight itself

- [Official Midnight documentation](https://docs.midnight.network/)
- [Compact language guide](https://docs.midnight.network/compact/writing)

## Reporting a security issue

Do not open a public issue. See [SECURITY.md](SECURITY.md) for the private disclosure process.
